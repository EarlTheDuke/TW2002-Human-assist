Fix net_worth to count equipment + owned planets

Victory scoring bug: time_net_worth was ranking Captain Reyes at 24.2k
credits despite owning a Citadel-L1 planet with ~1000 colonists ferried
to it (15,000 cr of sunk investment + 10,000 cr of colonist pool value).
The old Player.net_worth property only summed credits + FO/Org/Eq cargo
+ ship hull + fighters -- everything else was invisible.

What now counts in net worth:

  Ship-side (Player.net_worth property, still zero-arg for backward compat):
    * credits
    * all cargo (FO/Org/Eq at base prices, colonists at 10cr/head)
    * ship hull at 50% resale
    * fighters, shields (10cr/unit), armid/limpet/atomic mines,
      photon missiles (12k), ether probes (5k), genesis torpedoes (25k)

  Planet-side (new full_net_worth(universe, player) helper):
    * ship-side, plus for every planet this commander owns:
      - citadel investment: cumulative CITADEL_TIER_COST for levels 1..N
        including the 10cr/colonist cost of consumed colonists
      - colonist pools (idle + productively assigned) at 10cr/head
      - commodity stockpile at base prices
      - planet treasury
      - planet defense (fighters + shields)

Call sites updated to use full_net_worth:
  * engine/runner.py victory checks (elimination AND time_net_worth)
  * engine/observation.py (what the LLM sees)
  * server/runner.py snapshot + state patch + history sampler

Victory GAME_OVER event now includes net_worth_ship, net_worth_planets,
and all_worths breakdown so spectators can see "X won on 50k ship +
35k planets" instead of one opaque number.

Server snapshot adds net_worth_ship alongside net_worth so the UI can
show "40k +15kp" (40k total, 15k of which is planet investment) with
full breakdown in tooltip. Gold-accented suffix for visual distinction.

Default max_days bumped 10 -> 15 (CLI serve and server create_app) to
give agents enough runway for a full economy arc: ship upgrade day 2-3,
first Genesis ~day 4-6, Citadel L2 ~day 8-10, L3 ~day 12-14.

Four new regression tests under TestPhaseGObservationSurface
(test_g5..g8) lock in: equipment valuation, planet asset counting,
isolation between players' planets, and the split-summary payload on
time_net_worth wins. 134/134 tests pass.
